## Install

You have two installers. Pick the one that matches your taste:

### Option A — `./install.sh` (default)

Pure-bash, no external dependencies. Backs up pre-existing files, renders
`condarc` with your real `$HOME` (since YAML can't expand it), and skips
kitty on headless machines.

```sh
git clone git@github.com:<you>/dotfiles.git ~/dotfiles
cd ~/dotfiles
./install.sh --dry-run     # preview
./install.sh               # do it
```

Flags:

- `--dry-run` — show what would happen
- `--no-conda` — skip `condarc` rendering (useful on cluster nodes)

### Option B — `./stow.sh`

Uses GNU Stow. Requires `stow` to be installed (`apt install stow`,
`brew install stow`, etc.). Faster, conventional, supports clean
`--uninstall`. Doesn't manage `condarc` (Stow can't template files).

```sh
./stow.sh --dry-run        # preview (uses stow --no)
./stow.sh                  # install all packages
./stow.sh zsh git tmux     # install only listed packages
./stow.sh --uninstall      # remove all symlinks
```

Both installers create `~/.gitconfig.local` and `~/.ssh/config.local`
from templates if missing, and never overwrite them on later runs.

After install, edit your identity:

```sh
$EDITOR ~/.gitconfig.local       # name + email
$EDITOR ~/.ssh/config.local      # personal/internal hosts
chmod 600 ~/.gitconfig.local ~/.ssh/config.local
```

### How the `stow/` tree stays in sync

The `stow/` directory is a tree of symlinks pointing back into the
real source files (`shell/`, `git/`, etc.). It's generated by
`build-stow-tree.sh` so you never maintain two copies of anything.
If you add or move a tracked file:

```sh
./build-stow-tree.sh                 # regenerate
./build-stow-tree.sh --check         # CI-friendly (exit 1 if stale)
git add stow/ && git commit
```

`stow.sh` runs `build-stow-tree.sh --check` and rebuilds automatically
if needed, so you can't get into a "stale tree" state at install time.

## How profiles work

The base `.zshrc` ends with a profile dispatch:

```sh
DOTFILES_PROFILE="${DOTFILES_PROFILE:-personal}"
# Search overlay first, then base
for _base in "${DOTFILES_OVERLAY:-}" "$DOTFILES"; do
    [[ -z "$_base" ]] && continue
    _f="$_base/shell/profiles/${DOTFILES_PROFILE}.zsh"
    [[ -r "$_f" ]] && { source "$_f"; break; }
done
```

So:

- **Personal machine**: nothing to do. `DOTFILES_PROFILE` defaults to
  `personal` and `profiles/personal.zsh` from this repo loads (kitty,
  fastfetch, conda, mamba, julia, uv).
- **Cluster machine**: a separate private repo (`dotfiles-cluster`)
  drops `profiles/cluster.zsh` into its own tree, and machines set:

  ```sh
  # in ~/.zshrc.local
  export DOTFILES_OVERLAY="$HOME/dotfiles-cluster"
  export DOTFILES_PROFILE="cluster"
  ```

  The base `.zshrc` then loads the cluster profile from the overlay
  instead of the personal one. The cluster profile in turn dispatches
  on hostname for site-specific bits (CC-IN2P3 vs IFIC, etc.).

You can drop ad-hoc snippets in `~/.config/zsh/env.d/*.zsh` — they're
sourced automatically near the top of `.zshrc`. Useful for one-off tools
that don't justify their own profile entry.

## Secret handling

What's never in this repo:

- Real name, email, usernames, internal hostnames → live in `*.local`
  files (gitignored).
- API keys, SSH private keys, GPG keys, keytabs, `.env` files →
  blocked by `.gitignore`.

Before pushing, audit with [gitleaks](https://github.com/gitleaks/gitleaks):

```sh
gitleaks detect --source . --verbose
```

For secrets that genuinely need to live in a repo, use
[`sops`](https://github.com/getsops/sops) with `age`, or
[`git-crypt`](https://github.com/AGWA/git-crypt). Don't commit them
plaintext.

## Notes & quirks

- **`ZDOTDIR`** is set to `$XDG_CONFIG_HOME/zsh` in `.zshenv`, so
  `.zshrc` / `.zprofile` live there, not in `$HOME`.
- **`compdef`** must never be called before `compinit`. That means no
  `eval "$(uv generate-shell-completion zsh)"` or similar in `.zshenv`
  / `.zprofile`. They go in `profiles/personal.zsh` after `.zshrc`'s
  `compinit`.
- **`zoxide`** is bound to `j`, not `cd`. `cd` keeps standard semantics.
- **Tmux auto-attach** (in kitty) can be disabled with `NO_TMUX=1 kitty`.
- **YAML doesn't expand `~` or `$HOME`**, so `condarc` paths are
  templated with `__HOME__` and rendered by `install.sh`.
